OpenAI เผยแพร่เอกสารเชิงเทคนิคหายากที่อธิบายวิธีที่ทีมวิศวกรรมออกแบบสถาปัตยกรรม WebRTC ใหม่ทั้งหมดเพื่อรองรับ Voice AI ให้ผู้ใช้กว่า 900 ล้านคนต่อสัปดาห์ เงื่อนไขสำคัญสามข้อที่ต้องบรรลุพร้อมกันคือ การครอบคลุมระดับโลก การเชื่อมต่อทันทีที่เริ่มพูด และ round-trip time ต่ำเสถียรไม่มี jitter สถาปัตยกรรมใหม่ใช้โมเดล "split relay + transceiver" ที่แยก edge service รับสัญญาณจากผู้ใช้แล้วแปลงเป็น internal protocol ที่เบากว่า แก้ปัญหา stateful ICE/DTLS sessions ที่เดิมต้องผูกติดกับ server เดิมตลอดเวลา

เอกสารนี้หายากมากจาก OpenAI ที่ปกติไม่เปิดเผยรายละเอียด production infrastructure และกลายเป็น benchmark ใหม่สำหรับวงการ real-time AI ในเชิงเทคนิค การจัดการ stateful sessions ในระดับ 900M users คือ challenge ที่ทีมส่วนใหญ่ยังไม่เคยเจอ นักพัฒนาที่สร้าง voice agent ควรศึกษาสถาปัตยกรรมนี้โดยเฉพาะวิธีแยก edge connection ออกจาก model inference layer — pattern นี้ช่วยลด latency ได้จริงและสามารถนำไปประยุกต์กับ WebRTC stack ของตัวเองได้เลย
