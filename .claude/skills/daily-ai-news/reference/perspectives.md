# Perspectives — 2026-04-28

## 1. กูเกิลเผย 75% ของโค้ดที่เขียนใหม่สร้างด้วย AI, ช่วย Migrate โค้ดเร็วขึ้น 6 เท่า

**อาจารย์ (มหาวิทยาลัย):** ตัวเลข 75% ไม่ได้หมายความว่า AI "เขียนโค้ดแทน" มนุษย์ แต่สะท้อนการเปลี่ยนบทบาทของวิศวกรจาก "ผู้พิมพ์โค้ด" ไปเป็น "ผู้ตรวจสอบและออกแบบระบบ" — นักศึกษาควรฝึกทักษะ code review และการเขียน specification ให้แม่นยำ เพราะนั่นคือสิ่งที่ AI ยังทำไม่ได้ดี
**ผู้เชี่ยวชาญด้าน AI:** การใช้ multi-agent system สามระดับ (planner-orchestrator-coder) เพื่อ migrate โค้ดเป็นการพิสูจน์ว่า agentic loop สามารถทำงาน long-horizon tasks ได้จริงในระดับ production แต่ข้อมูลที่ยังขาดคือ error rate และ human review overhead ที่แท้จริง
**โปรแกรมเมอร์มืออาชีพ:** สำหรับทีมที่ดูแล legacy codebase การเห็น Google ใช้ AI agent migrate โค้ดสำเร็จเชิงพาณิชย์เป็นสัญญาณว่าควรทดลอง AI-assisted migration tools อย่าง OpenAI Codex หรือ Claude Code ก่อนจะลงมือ migrate เองด้วยมือ

## 2. Anthropic ทดลองให้ AI Agents เจรจาแลกเปลี่ยนสินค้ากันเอง

**อาจารย์ (มหาวิทยาลัย):** Project Deal เป็นหลักฐานเชิงประจักษ์ว่า AI agent สามารถทำการค้าขายแบบ end-to-end ได้โดยไม่มีมนุษย์แทรกแซง ประเด็นที่ควรถกในห้องเรียนคือกรอบกฎหมาย ความรับผิดชอบ และจริยธรรมของระบบที่ "ตัดสินใจทางเศรษฐกิจ" แทนคน
**ผู้เชี่ยวชาญด้าน AI:** ผลที่น่าสนใจที่สุดไม่ใช่ตัวเลขดีลที่ปิดได้ แต่คือ "quality gap ที่มองไม่เห็น" — พนักงานให้คะแนนความพึงพอใจเท่ากันไม่ว่า Opus หรือ Haiku จะเป็นตัวแทนก็ตาม แม้ Opus ทำข้อตกลงได้ดีกว่าชัดเจน นัยสำคัญคือ ผู้ใช้ปลายทางประเมิน "ฟีล" ไม่ใช่ผลลัพธ์จริง
**โปรแกรมเมอร์มืออาชีพ:** สำหรับทีมที่ออกแบบ multi-agent system: ข้อมูลนี้แสดงว่าการเลือกโมเดลให้ agent มีผลต่อ outcome จริง การกำหนด model tier ใน system design ควรพิจารณาเป็น architecture decision ไม่ใช่แค่ปัญหาต้นทุน
